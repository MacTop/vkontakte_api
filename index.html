Что это?

vkontakte_api - ruby-обертка для ВКонтакте API. Она позволяет вызывать методы API и получать их результаты в виде Hash и Array.

Методы из пространств имен (например, friends.get) реализованы в виде цепных вызовов, например app.friends.get(fields: ['last_name', 'first_name']).

Как этим пользоваться?

Можно использовать vkontakte_api как для авторизованных запросов (требующих авторизованного пользователя, типа friends.get или groups.get без параметра uid), так и для неавторизованных (типа users.get).

Неавторизованные запросы

Если все, что нужно получить от вконтакта - это публично доступные данные, можно пойти по простому пути:

require 'vkontakte_api'
@app = VkontakteApi::Client.new
@app.users.get(uid: 1)
# => [
#     [0] {
#                :uid => 1,
#         :first_name => "Павел",
#          :last_name => "Дуров"
#     }
# ]

Авторизованные запросы

Но если есть необходимость вызывать какие-то методы, требующие авторизации пользователя, то придется получить токен доступа.

Авторизация

Схема получения токена доступа следующая:

  • редиректим пользователя на ВКонтакте, там он подтверждает запрошенные права доступа
  • ВК редиректит его обратно в наше приложение, передавая в параметрах код
  • запрашиваем у ВК токен доступа, используя код, полученный шагом ранее
  • с токеном можно вызывать методы ВКонтакте API

Первое, что нужно сделать - зарегистрировать приложение на vk.com. Идем сюда, вводим название приложения, его домен и тип. После создания на странице редактирования приложения можно найти ID приложения и защищенный ключ.

Далее перебрасываем пользователя на ВК для получения прав на доступ к его данным. URL для редиректа составляется по следующей схеме:

http://oauth.vk.com/authorize? 
 client_id=APP_ID& 
 scope=SETTINGS& 
 redirect_uri=REDIRECT_URI& 
 response_type=code

где APP_ID - ID приложения, SETTINGS - права доступа, которые мы хотим получить (полный список возможных прав можно почитать здесь), а REDIRECT_URI - урл приложения, куда пользователь будет переброшен после подтверждения прав доступа.

При редиректе пользователя обратно в приложение будет передан параметр code:

http://REDIRECT_URI?code=7a6fa4dff77a228eeda56603b8f53806c883f011c40b72630bb50df056f6479e52a

Теперь, когда у нас есть код, можно получить токен доступа. Выполняем второй запрос на следующий URL:

https://oauth.vk.com/access_token? 
client_id=APP_ID& 
client_secret=APP_SECRET& 
code=7a6fa4dff77a228eeda56603b8f53806c883f011c40b72630bb50df056f6479e52a

Тут мы используем ID приложения (APP_ID) и защищенный ключ (APP_SECRET) из настроек приложения, а также код из предыдущего шага. Ответ придет в формате JSON:

{"access_token":"533bacf01e11f55b536a565b57531ac114461ae8736d6506a3", "expires_in":43200, "user_id":6492}

На этом авторизация завершена, и можно пользоваться API.

Использование API

# создаем клиент API, ассоциированный с токеном доступа
@app = VkontakteApi::Client.new('533bacf01e11f55b536a565b57531ac114461ae8736d6506a3')

# и выполняем запросы
@app.groups.get # => [1, 31022447]
@app.friends.get(fields: ['uid', 'first_name', 'last_name'])
# =>  [
#       {
#         :uid          => "1",
#         :first_name   => "Павел",
#         :last_name    => "Дуров"
#       },
#       {
#         :uid          => "6492",
#         :first_name   => "Andrew",
#         :last_name    => "Rogozov"
#       }
#     ]
